# Seedpeak Workflow

种子驱动的工作流程，将模糊灵感转化为可执行的目标和计划。

## 触发条件

当用户输入匹配以下模式时激活：
- `种子：<模糊灵感>` / `seed: <idea>`
- 用户表达一个模糊想法并希望结构化推进

## 流程概览

```
1. 明确目标 → 2. 制定方案 → 3. 补充上下文 → 4. 执行 → 5. 整合 → 6. 反馈 → 7. 迭代
```

## 核心原则

1. **逐步收敛** — 从模糊到清晰，一次只问 1-2 个维度
2. **方案先行** — 执行前必须完成目标、方案、上下文的确认
3. **递归分解** — 复杂子任务单独应用此 7 步流程
4. **主动暴露风险** — 不要执行到一半才说"这个做不了"

## 步骤说明

### 1. 明确目标
收到种子后，生成维度清单 → 逐项赋值 → 汇总为最终目标

### 2. 制定方案
任务拆分 → 执行次序 → 技术方案 → 审核方案

### 3. 补充上下文
确认角色/背景/输出要求/参考资料/禁忌

### 4. 执行
按方案执行，复杂子任务递归应用此流程

### 5. 整合
将所有产出整合为统一产物

### 6. 反馈
自检 + 邀请用户评价

### 7. 迭代
根据反馈持续改进，形成闭环
